Batmaste replied on 31-Oct-07 12:48 PM
I finally found a place that told me how to calculate the CPU.
http://www.alexfedotov.com/samples/wmitop.asp
I hope this helps someone else with their search to finding a way to
get the CPU without Performance Counters.

Using Win32_Process, You are able to get the UserModeTime and the
KernelModeTime which allows you to calculate the CPU.

UserTime = ((New_UserModeTime - Old_UserModeTime) / TimeInterval) *
100;

KernelTime =(( New_KernelModeTime - Old_KernelModeTime) /
TimeInterval) * 100;

CPU = (((New_UserModeTime - Old_UserModeTime) + (New_KernelModeTime -
Old_KernelModeTime)) / TimeInterval) * 100;

NOTE: When calculating CPU do not use the UserTime and
KernelTime calculations.

Benjamin Batman replied on 08-Feb-09 10:47 AM
Hi, Doris



You are correct about the TimeInterval, it is the time in between sampling of the Process's info. I used C# when I was using this, and stored Datetime.Now, then when I compared the New and Old info for that process, I just did a subtracted the Old from the New.



The units of UserModeTime and KernelModeTime are ulong, which is a 64-bit unsigned integer



Also if the computer you are running this on has more than one processor you will have to divide by the number of processors (CPU = CPU / NumProcessors). If you are using C# you can use Environment.ProcessorCount to get the number of processors.



I hope this helps,

One thing about using WMI, that might or might not affect you, is that is can be a bit slow in retrieving the information.
